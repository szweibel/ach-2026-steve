tools.ailab.gc.cuny.edu

The Tools Portal is where the lab puts standalone apps, separate from the chat Sandbox Zach showed you. Some handle media — transcription in dozens of languages, OCR for archival scans, alt-text for accessibility. There's a disclosure tool that gives a scholar a citable record of which tasks they handed to a model. The ones I want to dwell on are the agentic studios. All of it runs on zero-retention paths.

Working with agentic AI

  • Agent Studio and Site Studio run agents for research and publishing
  • People use them for research and scholarly-communication work
  • It takes several steps to finish a task on its own
  • Faculty and students build working tools with little code
The studios are how we put agentic AI in people's hands — not a chatbot you ask, but an agent you hand a task to. That shift, from asking to delegating, is what's reshaping research workflows and scholarly communication. Someone can build a small research tool or an academic site without writing every line, and learn what it means to work alongside an agent.

Teaching librarians to build with AI

Agentic AI for Library Practice, a 16-week program

  • For CUNY librarians, no programming background needed
  • Build working tools, like Primo dashboards and LibGuides interfaces
  • Use Claude Code, then investigate what it built
  • Spot hallucinations and errors, and verify against the docs
  • They leave clearer on what AI can do and how to work with it
  • Better equipped for their work with patrons and students across CUNY
This is a program I lead, Agentic AI for Library Practice. It runs sixteen weeks, for CUNY librarians with no programming background. They use a coding agent to build working tools for library problems, like collection dashboards on the Primo API or custom LibGuides interfaces. They're learning to investigate the agent's output. They trace the files and check its claims against documentation, so they catch the hallucinations. They come out able to build their own tools, and clearer on what AI can do and how to work with it. They are better equipped for their everyday work with patrons and students across CUNY.

Kale Workbench

a coding agent in the browser

  • Like Claude Code or Codex, run in the browser
  • On the lab's Cloudflare infrastructure, with open models
  • The lab runs it, so it sets the model and the limits
  • Nothing gets pushed or published without your approval
  • It hands off to Kale Deploy when a project is ready
Kale Workbench is a web-based coding agent, the kind of thing Claude Code or Codex do in a terminal. The lab runs it, on Cloudflare, with open models by default. Because it is ours, we set the model and the spending limits, and the agent can't push to GitHub or publish without you. When a project is ready, it hands off to Kale Deploy. (Active repo, roadmap maturity.)

Kale Deploy

live hosting at a CUNY address

  • Agents deploy it themselves, through an MCP server
  • Build with an agent, and it's live at <project>.cuny.qzz.io
  • The first place CUNY students and faculty can publish live projects
  • It hosts many projects at once for very little
  • Users run their own projects
Kale is an MCP server, so a coding agent can deploy a repo and check the result on its own. Build something with an agent and it's live at a CUNY address. CUNY students and faculty haven't had a place to put live, working projects like this before. And because it runs on Cloudflare's Workers for Platforms, the lab can host many projects at once for very little, and hand the running of each one to the person who built it.
